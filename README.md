# Codebase Documenter

A Claude Code plugin that generates and maintains comprehensive codebase documentation optimized for AI consumption.

## Installation

Inside Claude Code, run:

```
/plugin marketplace add wondersauce/claude-code-plugin
 
/plugin install ws-codebase-documenter@wondersauce-marketplace
```

## Usage

Once installed, ask Claude Code to document your codebase:

```
/ws-codebase-documenter bootstrap
```

For incremental updates after code changes:

```
/ws-codebase-documenter update
```

To regenerate a specific document without a full re-run:

```
/ws-codebase-documenter regenerate playbook
/ws-codebase-documenter regenerate style-guide
/ws-codebase-documenter regenerate all
```

Valid regenerate targets: `playbook`, `capability-map`, `style-guide`, `integration-map`, `overview`, `architecture`, `consistency-report`, `all`

## What It Does

**Bootstrap mode** (first run):
- Detects your stack (Node.js, Python, Go, Rust, .NET, Java, PHP)
- Creates `documentation/config.json` with auto-detected settings
- Scans for public/private APIs, frontend assets, cross-module patterns
- Generates structured markdown docs including prescriptive guides
- Injects project-specific rules into `CLAUDE.md`

**Incremental mode** (subsequent runs):
- Analyzes git diff since last run
- Updates only changed functions/types
- Keeps prescriptive docs (playbook, capability map, style guide, integration map) in sync
- Runs consistency checks against established patterns
- Flags violations in PR descriptions

**Regenerate mode** (targeted re-generation):
- Re-scans only what's needed for the specified document
- Regenerates a single doc without advancing the incremental baseline
- Useful when a generated doc has inaccurate patterns or needs a reset

### Prescriptive Documents

| Document | Purpose |
|----------|---------|
| `playbook.md` | Step-by-step procedures for adding endpoints, models, components, etc. Prevents architectural drift. |
| `capability-map.md` | "I need to do X вЂ” what should I use?" lookup. Prevents code duplication. |
| `style-guide.md` | CSS methodology, design tokens, JS patterns, forbidden anti-patterns. Prevents frontend entropy. |
| `integration-map.md` | Module dependency matrix, hook/event contracts, shared resources. Prevents cross-module inconsistency. |

Each document is generated by analyzing detected patterns in your codebase and can be toggled via `config.json`.

## Docusaurus Sync (Optional)

To sync docs to a separate Docusaurus site, add to `documentation/config.json`:

```json
{
  "docusaurus": {
    "repo": "git@github.com:your-org/docs-site.git",
    "branch": "main",
    "docs_path": "docs/api",
    "sidebar_label": "API Reference"
  }
}
```
